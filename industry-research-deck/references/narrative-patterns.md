# 叙事框架库（L2，按需展开）

拆段时**可选**参考。每个框架附适用判断 — 选合适的，别硬套。

## 1. Kurzgesagt 5 段式（最经典，默认推荐）

```
[1] 钩子：一个反常识的现象 / 数字       (15-20%)
[2] 这到底是什么？拆开概念                (15-20%)
[3] 为什么会这样？（机制 / 原因）         (25-30%)
[4] 这意味着什么？（后果 / 受影响的人）   (20-25%)
[5] 接下来呢？（未来 / 展望 / 收束）     (10-15%)
```
**适合**：科普、原理解释、为啥 X 火 / 为啥 Y 不行。

## 2. Vox 3 幕式（短平快）

```
[1] 问题（What's happening）              30%
[2] 解释（Why it matters）                40%
[3] 收束（Where it goes）                 30%
```
**适合**：60s 以内的极短讲解；不需要太多铺垫的"硬新闻"。

## 3. Bloomberg 数据叙事

```
[1] 反差数字开场（震一下）
[2] 同类对比（vs 历史 / vs 同业 / vs 预期）
[3] 关键玩家受益排序（榜单）
[4] 风险 / 反方观点（平衡）
[5] 一句话总结（趋势判断）
```
**适合**：财经、产业涨跌、公司业绩。

## 4. Stratechery 战略分析

```
[1] 起因（导火索）
[2] 结构性原因（更深一层）
[3] 玩家博弈（who wins / who loses）
[4] 长期影响（5-10 年视角）
```
**适合**：技术战略、产业格局变化、政策影响。

## 5. 故事弧（人物驱动）

```
[1] 主角与挑战
[2] 转折点
[3] 关键决定
[4] 结果与代价
[5] 启示
```
**适合**：公司兴衰、人物传记、案例研究。

## 6. 问题树（MECE 拆解）

```
[1] 现象
[2] 因 A
[3] 因 B
[4] 因 C
[5] 综合 / 主因
```
**适合**：多因素并存的复杂问题；不适合 2 分钟以内（信息密度太高）。

---

## 选用判断

| 主题特征 | 推荐框架 |
|---|---|
| 解释"为啥 X 火 / 为啥 Y 失败" | 1 Kurzgesagt |
| 财经数据 / 业绩 / 涨跌 | 3 Bloomberg |
| 短平快新闻解读 | 2 Vox |
| 战略 / 产业格局 | 4 Stratechery |
| 公司或人物故事 | 5 故事弧 |
| 多因素复杂分析 | 6 问题树 |

不在表里的，自己组合或新造。

## 反模式（不要做）

- ❌ 每段都从"首先 / 其次 / 然后"开始（叙事节奏会死掉）
- ❌ 数据堆砌不解释（观众记不住）
- ❌ 末段强行总结所有要点（应该收束到**一个**判断或预测）
- ❌ 钩子用陈词滥调（"你知道吗"、"今天我们来聊聊"）
